### 🐧 Linux Filesystem Hierarchy

---


# 1. 🧭 INTRODUCTION

Linux filesystem is a **single-root hierarchical structure** governed by the Filesystem Hierarchy Standard (FHS).

Unlike Windows:

* No drive letters
* Everything is mounted under `/`
* Devices are abstracted as files

---

# 2. 📜 FILESYSTEM HIERARCHY STANDARD (FHS)

FHS defines:

* Directory structure
* File placement rules
* Compatibility across distributions

Key goals:

* Predictability
* Portability
* System consistency

---

# 3. 🌳 ROOT FILESYSTEM DEEP DIVE

```
/
├── bin
├── boot
├── dev
├── etc
├── home
├── lib
├── media
├── mnt
├── opt
├── proc
├── root
├── run
├── sbin
├── srv
├── sys
├── tmp
├── usr
└── var
```

---

# 4. 📂 DIRECTORY ANALYSIS

## 🔹 /bin (Essential Binaries)

Critical user commands required during boot:

Examples:

```
ls, cp, mv, bash, cat
```

Notes:

* Required in single-user mode
* Symlinked to `/usr/bin` in modern systems

---

## 🔹 /boot

Contains:

* Kernel (vmlinuz)
* initramfs
* GRUB bootloader files

Example:

```
/boot/grub2/grub.cfg
```

---

## 🔹 /dev (Device Files)

Represents hardware as files:

Types:

* Block devices → disks
* Character devices → terminals

Examples:

```
/dev/sda
/dev/null
/dev/tty
```

---

## 🔹 /etc (Configuration)

System-wide configs:

Examples:

```
/etc/passwd
/etc/shadow
/etc/fstab
/etc/ssh/sshd_config
```

---

## 🔹 /home

User directories:

```
/home/user1
/home/admin
```

---

## 🔹 /var

Dynamic data:

Subdirs:

* `/var/log`
* `/var/lib`
* `/var/cache`

---

## 🔹 /usr

Secondary hierarchy:

```
/usr/bin
/usr/lib
/usr/share
```

---

## 🔹 /proc (Virtual FS)

Kernel interface:

```
/proc/cpuinfo
/proc/meminfo
```

---

## 🔹 /sys

Hardware interaction layer

---

## 🔹 /tmp

Temporary files:

* World writable
* Often cleared on reboot
